\question[2] Escreva uma função interativa e uma função recursiva para calcular a multiplicação de números naturais.

\begin{eqnarray*} 
  \text{Uma definição interativa de $a \ * \ b$ é:} \ a \ * \ b \ = \left\{ 
    \begin{array}{ll}
      a, \ se \ b \ = \ 1,      \\
     \underbrace{a + a + a + \cdots + a}_{b \ vezes}, \ se \ b \ > \ 1.            
    \end{array} \right.     
\end{eqnarray*}


\begin{eqnarray*} 
  \text{Uma definição recursiva de $a \ * \ b$ é:} \ a \ * \ b \ = \left\{ 
    \begin{array}{ll}
      a, \ se \ b \ = \ 1,      \\
     a \ * \ (b - 1) + a,  \text{se $b \ > \ 1$}
    \end{array} \right.     
\end{eqnarray*}

\question[1] Sobre os uso das funções recursivas, assinale a(s) alternativa(s) \textbf{CORRETA(S)}.
\begin{parts} 
  \part (\hspace{0.3cm}) Impede que se garanta a finalização de um algoritmo.
  \part (\hspace{0.3cm}) Em geral, demanda menos memória que as alternativas iterativas, pois há apenas
um escopo para as variáveis nas diversas chamadas.
  \part (\hspace{0.3cm}) Pressupõe a definição do problema a ser resolvido em instâncias menores do mesmo problema.
  \part (\hspace{0.3cm}) É sempre mais eficiente do que as alternativas baseadas em iteração.
  \part (\hspace{0.3cm}) Uma estrutura recursiva é aquela que consiste, parcialmente, ou é definida em função dela mesmo.
  \part (\hspace{0.3cm}) Todo algoritmo recursivo corresponde a um outro não recursivo que executa, exatamente, o mesmo trabalho computacional.
\end{parts}

\question[2\half] Para um dado número inteiro $n > 1$, o menor inteiro $d > 1$ que divide $n$ é chamado de \textbf{fator primo}. É possível determinar a fatoração prima de $n$ achando-se o fator primo $d$ e substituindo $n$ pelo quociente $n / d$, repetindo essa operação até que $n$ seja igual a 1. Utilizando uma das estrutura de dados estudadas em sala para auxiliá-lo na
manipulação de dados, implemente um algoritmo que compute a fatoração prima de um número apresentando os seus fatores em ordem decrescente. Por exemplo, para $n = 3960$, deverá ser apresentado $11 * 5 * 3 * 3 * 2 * 2 * 2$. Justifique a escolha da estrutura de dados utilizada.

\question[2] Uma empilhadeira carrega caixas de 3, 5, e 7 toneladas. Há três pilhas A, B e C. A pilha A é onde se encontram as todas as caixas que chegam no depósito. Com um detalhe: caixas maiores não podem ser empilhadas sobre caixas menores. Elabore um algoritmo que efetue o controle das caixas de forma que caso uma caixa de maior peso do que uma que já está em A deva ser empilhada, todas as caixas que estão em A são movidas para as pilhas auxiliares B (contendo somente caixas de 5 toneladas) e C (contendo somente caixas de 3 toneladas) até que se possa empilhar a nova caixa. Depois, todas as caixas são movidas de volta para a pilha A.
